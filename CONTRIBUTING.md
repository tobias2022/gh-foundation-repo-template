# Contributing Guide

Thank you for contributing.

## Branching
- Create feature branches from the default branch.
- Use descriptive branch names, for example: `feature/add-api-health-check`.

## Pull Request Requirements
- Use the pull request template.
- Keep pull requests focused and reasonably sized.
- Link related work items or issues.
- Add or update tests where applicable.
- Update documentation when behavior changes.

## Review Expectations
- At least one approval is required before merge.
- Changes to code-owned files require code owner approval.
- All required checks must pass.

## Commit Guidance
- Use clear, imperative commit messages.
- Keep commits logically grouped.

## Coding and Quality Standards
- Follow language- and project-specific style guides.
- Avoid committing secrets or sensitive data.
- Ensure local validation passes before opening a pull request.

## Reporting Issues
- Use the repository issue templates if available.
- Include reproducible steps and expected versus actual behavior.
